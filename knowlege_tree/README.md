>###### 测试分析设计方法

>* 测试设计方法

>    * 判断表/因果图
>      * 检查业务规则,针对业务规则比较复杂
>      * 控制比较多和复杂的情况,比如游戏
>    * 正交试验
>      * 检查输入组合
>      * 高级查询,组合查询
>    * 流程分析
>      * 检查业务流程
>    * 状态迁移
>      * 检查工作状态的变化
>    * 错误猜测
>      * 基于经验检查,靠经验积累和用户反馈
>      * 查看缺陷库
